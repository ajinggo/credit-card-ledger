# 手续费汇总显示设置设计

## 目标

在“手续费汇总”的记录明细工具栏增加统一的“显示设置”，让用户分别控制顶部汇总指标和明细表格列的显示状态。设置应立即生效，并随本机设置、云端账本快照和完整 JSON 备份保存。

## 界面

- 在“筛选”旁增加“显示设置”按钮，按钮负责打开和关闭轻量浮层。
- 浮层分为“汇总指标”和“记录明细列”两组复选框。
- 汇总指标包括：筛选记录、套现总额、手续费、净收益。
- 明细列包括：日期、信用卡、消费形式、渠道、套现金额、手续费、费率、积分价值、净收益、账单月份、操作。
- 所有项目默认显示，浮层底部提供“恢复默认”。
- 点击浮层外部或按 `Escape` 关闭，按钮通过 `aria-expanded` 表达状态。

## 行为规则

- 勾选变化立即应用，不需要刷新或额外保存。
- 汇总指标全部隐藏时，整个汇总条收起，不保留空白。
- 明细数据列至少保留一项；“操作”列不计入这个限制。
- 隐藏列只影响页面展示，不改变筛选、排序、CSV 导出或原始记录。
- 新增或编辑记录后继续沿用当前显示设置。
- 窄屏使用同一浮层，并保持表格横向滚动能力。

## 数据结构

使用一个版本化设置对象：

```js
{
  summary: ["count", "spend", "fee", "net"],
  columns: ["date", "card", "method", "channel", "cashout", "fee", "rate", "pointValue", "net", "billMonth", "actions"]
}
```

- 本机存储使用独立的 `localStorage` 键。
- 读取时只接受已知标识，忽略未来版本或损坏数据中的未知值。
- 未保存设置或设置无效时回退为全部显示。
- 设置加入账本快照的 `settings`，因此云同步和 JSON 导入导出使用同一份数据。

## 代码边界

- `index.html`：显示设置按钮、浮层、复选框和表头列标识。
- `app.js`：设置规范化、持久化、显示应用、菜单交互和快照读写。
- `organic-liquid.css`：浮层布局、复选框状态、隐藏规则和响应式样式。
- 独立模型模块负责默认值、规范化和至少保留一列的规则，便于无浏览器单元测试。

## 异常处理

- 损坏的本机设置不会阻止页面启动，直接回退默认值。
- 云端旧快照没有该字段时保持全部显示。
- 导入设置试图隐藏全部明细数据列时，恢复一个默认核心列。

## 验证

- 单元测试覆盖默认值、未知标识过滤、至少保留一项和不修改输入。
- DOM 回归测试覆盖按钮、两组复选框、列标识、快照字段和缓存版本。
- 手动检查桌面与手机宽度下的浮层定位、表格隐藏效果、恢复默认和页面刷新后的持久化。
- 完整运行现有共享额度、排序、性能、提示和记录卡片布局测试。
