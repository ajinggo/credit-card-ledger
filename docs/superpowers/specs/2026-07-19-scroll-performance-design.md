# 全站滚动性能优化设计

## 背景

当前页面在卡片、手续费、看板、积分和抽屉中滚动时都有明显卡顿。代码排查确认：

- 页面没有持续监听 `scroll`、`wheel` 或 `touchmove`，滚动期间不存在脚本反复计算。
- 卡片、账单、积分账户和手续费记录会一次性生成全部 DOM；数据越多，页面绘制面积越大。
- 玻璃主题经过多轮样式叠加，已有性能层关闭了部分模糊，但仍需要一个位于最终样式末尾的统一覆盖层，保证固定层、提示层、抽屉和重复内容不再产生昂贵的透明合成。
- 重复项目没有屏幕外延迟绘制和绘制隔离，浏览器仍需维护较大的布局与绘制范围。

本次采用已确认的 A 方案：均衡性能优化。保留全部记录和现有操作方式，不引入分页或“加载更多”。

## 目标

1. 降低所有主视图在拖动页面时的绘制和合成成本。
2. 数据较多时，屏幕外的卡片、账单、积分账户和手续费记录延迟到接近视口时再绘制。
3. 保留当前绿白视觉、固定手机导航、全屏抽屉和桌面布局。
4. 不改变任何财务口径、数据结构、Supabase 登录同步、备份导入导出和筛选统计行为。
5. 不增加分页，不隐藏记录，不改变记录排序或操作入口。

## 非目标

- 不重写应用框架。
- 不做虚拟列表或服务端分页。
- 不压缩、删除或迁移用户数据。
- 不调整信用卡、额度、账单、手续费、积分和还款的计算公式。
- 不在本次工作中改动 Supabase 表结构或权限策略。

## 根因与策略

### 1. 最终样式覆盖层

在 `organic-liquid.css` 最后增加 `Performance Pass V82`，使性能规则位于现有 V81 手机适配之后，避免旧主题规则重新覆盖优化。

覆盖范围包括：

- 顶部导航、底部导航和工具菜单。
- Dashboard 指标、手续费看板、图表、卡片、账单、积分账户和记录列表。
- Toast、筛选面板、所有抽屉、抽屉遮罩和抽屉内部固定标题/操作栏。
- 输入框、选择框和重复操作按钮。

这些常驻或可滚动表面统一关闭：

- `backdrop-filter`
- `-webkit-backdrop-filter`
- 大面积 `filter`
- `mix-blend-mode`
- 与滚动无关的持续动画和大范围过渡

背景改为接近不透明的现有绿白色，保持层级辨识度。大范围柔光阴影改为轻量边框和小阴影，不把界面改成纯平面。

### 2. 屏幕外延迟绘制

对以下重复内容使用 `content-visibility: auto`：

- `.card-row-wide`
- `.bill-row`
- `.loyalty-card`
- `.analytics-chart-block`
- 手续费记录行 `#recordsBody tr`

同时设置与现有布局相符的 `contain-intrinsic-size`，避免浏览器跳过屏幕外内容时滚动条长度突然变化。

手续费记录在手机端是网格列表，直接按行延迟绘制；桌面表格保留原有表格布局，并通过兼容性规则保证不支持该属性的浏览器仍按原方式显示全部内容。

### 3. 绘制隔离

对重复数据单元增加 `contain: paint style`，限制阴影、背景和状态变化的重绘范围。避免某张卡片、某条记录或某个按钮状态变化时扩大到整个页面。

不使用可能改变元素尺寸计算的 `contain: size`，也不对固定导航和抽屉容器使用布局隔离。

### 4. 固定层与抽屉

手机顶部栏和底部导航继续固定，但保持近不透明背景、无背景模糊和轻量边线。工具菜单、Toast 和抽屉使用相同原则。

抽屉内容区保留独立滚动，并增加：

- `-webkit-overflow-scrolling: touch`
- `overscroll-behavior: contain`

抽屉标题和底部操作栏继续固定，不改变现有全屏结构。

### 5. 页面切换滚动

`switchView()` 目前使用平滑滚动返回顶部。平滑动画可能与用户紧接着的拖动竞争，本次改为立即回到顶部：

```js
window.scrollTo({ top: 0, behavior: "auto" });
```

这只影响切换板块时的滚动动画，不影响视图、筛选、排序或数据。

## 文件范围

- `organic-liquid.css`：新增最终性能覆盖层和延迟绘制规则，缓存版本更新为 `v82`。
- `app.js`：板块切换改为立即滚动，构建标记和缓存版本更新为 `v74`。
- `index.html`：CSS/JS 缓存版本更新为 `v82` / `v74`。
- `tests/performance-budget.test.mjs`：在现有文件中补齐全部性能回归契约，不增加新的测试文件或运行依赖。

## 测试设计

### 自动化测试

先写失败测试，要求：

1. `Performance Pass V82` 必须位于 `Mobile Responsive V81` 之后。
2. 最终层明确关闭常驻表面、固定层、Toast 和抽屉的背景模糊。
3. 重复数据单元具有 `content-visibility: auto`、`contain-intrinsic-size` 和绘制隔离。
4. 抽屉滚动区保留惯性滚动和滚动边界控制。
5. `switchView()` 不再使用 `behavior: "smooth"`。
6. CSS 和 JS 缓存版本同步更新。

随后运行完整现有测试、JavaScript 语法检查和 Git 空白检查。

### 页面验证

验证尺寸：

- `360 x 800`
- `390 x 844`
- `430 x 932`
- `1440 x 900`

验证内容：

- 页面无横向溢出。
- 手机顶部栏和底部导航位置不变。
- 卡片、账单、积分账户和手续费记录在滚动前后尺寸稳定。
- 手续费记录的标签、卡名、积分数量和操作按钮完整显示。
- 添加卡片、登记账单、卡汇总、待办和备份抽屉仍覆盖整个手机视口。
- 桌面导航和工具栏布局不变。

## 风险与控制

### `content-visibility` 的浏览器差异

不支持该属性的浏览器会忽略它并保持当前完整渲染，不影响功能。`contain-intrinsic-size` 仅作为估算高度，不参与数据计算。

### 视觉层级变弱

通过边框、轻量阴影和接近不透明的背景保留层级，不移除分组结构，也不改变品牌颜色。

### 表格布局变化

性能规则不改变桌面表头、列宽、隐藏指标和排序结构；手机记录仍使用现有两列标签列表。

## 验收标准

- 所有现有功能和财务数据行为保持不变。
- 全部自动化测试通过。
- 四个目标视口无布局回归或横向溢出。
- 常驻滚动表面不再使用背景模糊或混合模式。
- 屏幕外重复项目具备延迟绘制和稳定占位。
- 本地预览可供用户直接比较优化前后的拖动手感。
