# 手续费看板密度与账单抽屉适配设计

## 目标

在浏览器 100% 缩放、常见桌面视口下，让手续费看板顶部双卡和完整六项汇总进入首屏，同时修复“卡汇总”和“登记月度账单”被固定导航遮挡、内容区域过大或操作按钮不稳定的问题。

## 已确认方向

采用视觉方案 A“完整汇总首屏”：

- 顶部总手续费卡和费率图形卡缩小约 30%。
- 六项手续费汇总完整保留并进入桌面首屏。
- 趋势分析保持现有可读尺寸，继续位于下一屏，不压缩成难以辨认的小图表。
- 不改变任何财务口径、筛选逻辑或数据结构。

## 手续费看板布局

桌面宽度下继续使用左右双卡布局，但压缩卡片的最小高度、内边距、主金额字号、装饰圆环和浮动指标。汇总区减少外层留白，并降低单项指标的最小高度；六项指标仍保持三列两行，筛选器仍位于标题右侧。

窄屏沿用现有响应式行为：隐藏装饰图形卡、汇总指标改为两列或单列。紧凑规则只影响手续费看板，不缩小“我的卡片额度”“手续费汇总”或“积分看板”的字号。

## 抽屉布局

“卡汇总”和“登记月度账单”使用同一套顶部安全区域：

- 遮罩从固定导航下方开始，高度为剩余可视区域。
- 抽屉高度限制为剩余可视区域，不再使用完整 `100vh`。
- 标题区固定在抽屉顶部，关闭按钮始终可见。
- 卡汇总表格独立滚动，标题和列头保持稳定。
- 登记账单表单内容独立滚动，保存与取消操作固定在底部。
- 桌面端卡汇总使用中等宽抽屉，账单表单使用更窄的输入型抽屉。
- 手机端两种抽屉均占满可用宽度，字段改为单列，并保留顶部与底部安全区域。

## 视觉约束

- 延续苹果系统字体和当前绿色、白色界面体系。
- 不增加新的卡片、动画或装饰图形。
- 抽屉圆角、边框、按钮样式与现有待办和数据工具保持一致。
- 关闭毛玻璃不必要的重复层，不重新引入滚动性能问题。

## 验收标准

- 1440 x 900、浏览器 100% 缩放时，顶部双卡和六项手续费汇总完整可见。
- 1280 x 720 时不出现横向溢出，核心金额和筛选控件不重叠。
- 390 x 844 时抽屉不被导航遮挡，所有字段和底部操作均可访问。
- 打开卡汇总和登记账单时，标题、关闭按钮、表头或保存按钮不会滚出可视区域。
- 桌面和手机截图无文字截断、控件重叠和不可用的滚动区域。
- 现有功能测试和性能回归测试全部通过。

## 测试范围

- 增加抽屉顶部偏移、剩余高度、固定标题和固定操作区的样式回归测试。
- 增加手续费看板桌面紧凑尺寸的样式回归测试。
- 使用真实浏览器分别检查 1440 x 900、1280 x 720 和 390 x 844。
- 验证卡汇总、登记账单的打开、关闭、滚动和保存交互。
